# MirrorLife Phase 1: Intuitive Core Loop Redesign

## Goal

New users should understand the core cause-and-effect loop within the first minute:

1. I put in one real-life fragment.
2. My avatar turns it into one visible action.
3. The city shows one explainable change.
4. I know what I can affect next.

This phase intentionally optimizes first-minute comprehension over world completeness.

## 3-Operation Core Loop Script

### Operation 1: Reality Fragment

Player sees:
- A focused first-minute panel, not the full tool menu.
- Input prompt: "刚刚发生了什么".
- Example placeholder: a short real conflict or pressure moment.

Player does:
- Types one concrete sentence, such as "今天和同事卡在一个分歧里，我不想再吵。"

System feedback:
- Step 1 becomes complete.
- Step 2, "分身行动", becomes the active decision.
- The simulation remains paused so ambient city motion does not steal attention.

Comprehension target:
- Player understands that their real-life fragment is the input, not a generic scenario generator.

### Operation 2: Avatar Action

Player sees three action choices:
- "先听懂" -> make expression clearer.
- "一起做小事" -> move pressure into a small collaboration.
- "先安抚" -> lower relationship tension first.

Player does:
- Clicks one action.

System feedback:
- The avatar performs that action immediately.
- The avatar and affected citizen are highlighted on canvas.
- A speech bubble and event entry confirm the action.
- The world remains slow/paused long enough to see the result.

Comprehension target:
- Player can say, "I chose what my avatar did."

### Operation 3: City Result + Next Choice

Player sees:
- A result card with one sentence explaining:
  - what the avatar did,
  - who or what changed,
  - which state changed,
  - what can be affected next.
- Buttons: "观察一回合" and "再做一次".

Player does:
- Either observes one turn of aftermath or tries a different action.

System feedback:
- The result is also written into the event stream and echo archive.
- Relationship lines and ambient city decoration become available only after this first causal loop is complete.

Comprehension target:
- Player can say, "The world changed because of my action, and I know my next lever."

## Minimal Playable Revision Boundary

Keep in Phase 1:
- One focused first-minute panel.
- One reality fragment input.
- Three plain-language avatar actions.
- One immediate city result card.
- One visible canvas highlight.
- One next-step choice.

Down-rank for Phase 1:
- Scene preset chips and scenario generation.
- Left-side feature menu.
- Bottom five-action toolbar.
- Event log as the primary feedback surface.
- Relationship lines and ambient animals before the first loop completes.
- Long-form worldbuilding explanations.

Do not add yet:
- More content categories.
- More NPC biographies.
- More action verbs.
- More tutorial text.
- Google Stitch visual production work; that starts after this logic script is accepted.

## Acceptance Checklist

- A new user can complete one loop without opening any side menu.
- Within three operations, the user sees input -> avatar action -> city result -> next choice.
- The first minute has one primary visual target.
- Ambient simulation is paused or slow until the first loop completes.
- The interface does not require understanding "镜像舱", "回声档案", or "人生剧本" before play.
