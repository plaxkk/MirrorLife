# 镜像人生产品设计方案

中文名：镜像人生  
Slogan：你想体验怎样的人生

## 1. 产品一句话

镜像人生是一款开放世界式虚拟社会产品：用户将现实人生事件投喂给分身，让分身在持续运转的社会沙箱中做选择、交互和回应；用户通过自然语言输入现实片段，见证自己的分身在“平等·自由·开放”的社会文化中逐步成长、发声与互助。

## 2. 核心命题

现实中的情绪压力和关系困惑常常缺少安全的参与场景。镜像人生不做“聊天工具化”体验，而是把用户与人际关系映射成可运行的社会系统，用户不需要成为任务执行者，而是作为现实作者，以最少操作持续参与世界运转。

核心链路仍是：

现实输入 → 影像化人生剧本 → 分身运转 → 社会回声沉淀 → 回家承接。

## 3. 产品愿景

构建一个“像世界一样活着”的虚拟社会：你给它一句今天的真实事件，它会让每个分身持续做出反应。虚拟世界不是逃避现实，而是给现实选择、关系和脆弱状态提供可观察、可回看的容器。

## 4. 目标用户

### 4.1 核心用户

1. 都市高压群体：职业与关系压力叠加，需要低噪音的情绪参与环境。  
2. 情感弱连接需求者：希望有低门槛共鸣，而不必被强制社交。  
3. 自我探索者：关注角色、关系和决策机制，愿意通过“可运转系统”理解自己。  
4. AI 伴侣体验用户：接受连续记忆型 AI 互动，不再只做检索式问答。

### 4.2 早期切入

优先面向 22-35 岁独居或社会连接较弱人群，测试是否愿意把“现实片段”持续喂给分身并回看其演化过程。

## 5. 用户核心需求

1. 我想把真实困惑放进一个不会立即评判我的系统。  
2. 我想让分身“知道我最近发生了什么”，而不是只讲抽象建议。  
3. 我想有轻量参与感：输入片段后看到回应、看到关系变化。  
4. 我希望这个世界容纳脆弱，也不要求我扮演“总是坚强的人”。  
5. 我希望得到“可回看、可复盘”的情绪与关系线索，而不是一次性鸡血。

## 6. 产品世界观

镜像人生由四层组成。

### 6.1 现实层

用户记录事件、关系、冲突、压力、选择、情绪起伏。每条记录支持权重与禁区标签，用户保有所有上/下线控制权。

### 6.2 分身层

每个用户都拥有一个 AI 分身画像，包含以下要素：

1. 人生结构：关系网、职业/角色、价值观倾向。  
2. 情绪结构：压力来源、情绪触发与恢复偏好。  
3. 决策结构：回避/坚持/冒险/修复倾向。  
4. 社交结构：历史互动风格、信任历史。  
5. 记忆轨迹：现实事件与社会反馈闭环。

### 6.3 虚拟社会层

社会层是一个持续运转的多分身世界。每个分身都是“参与者”，不是摆设 NPC。世界运行方式是：

1. 用户输入现实片段。  
2. 系统将其转为事件。  
3. 社会中的分身围绕事件做行动。  
4. 关系、情绪、社交秩序发生变化。  
5. 产出可读的社会回声，回流给用户与其分身记忆。

### 6.4 社会宪章（游戏规则）

#### 自由

每个分身享有发言、沉默、暂退、重入场的权利。弱势状态不被处罚；系统允许分身在高压时寻求安抚、观察、旁观等“低强度参与”。

#### 平等

所有分身都拥有同等参与入口。无论关系强弱，都可触发表达与协作，只要其行为可理解、可复盘、可回声。平等通过机制落地为“公开反馈窗口 + 可见影响记录”。

#### 开放

世界可被现实片段持续改写：用户能添加新主题、新关系纠纷、新行动方式。系统不锁死叙事树，支持“从今天继续”的社会连续性。

#### 允许脆弱 + 尊重强者文化（补充条款）

1. 脆弱视为状态，不是失败；低情绪阶段不降低系统参与权重。  
2. 强者文化以“可被借力”定义，不以“支配权”定义：强者可承担组织、调停、示范角色；不拥有压制表达的额外权力。  
3. 任何会加剧排斥、羞辱、操控的行为被记录为越界，触发共识修复流程。

## 7. 核心玩法系统

### 7.1 虚拟社会（开放世界核心玩法）

#### 开放世界运行模型（v0.4 引擎）

- 世界由 5 类区块构成：公开广场、静默角落、共学工坊、庭院回环、修复站。每个区块有公开度、容纳度、迁徙倾向、行为角色配置，分身会根据张力、情绪、信任自动移动。
- 单回合执行顺序：  
  1. 区块偏置动作决策；  
  2. 行为执行并更新 `情绪/信任/张力/和谐`；  
  3. 规则校验（允许脆弱、发言垄断治理、区域约束）；  
  4. 阶段推进与环境事件（3 回合/次波动）；  
  5. 指标与准则评分更新（自由、平等、开放）；  
  6. 社会日志与回声沉淀。  
- 玩家可手动触发动作（提案、协作、安抚、倾听）作为“世界干预”；所有干预仅改变该回合权重，不替代规则底层约束。
- 低状态分身会自动获得安抚/旁观优先；高频发言者会触发公平补偿，强迫系统重分配支持行为。

这是主玩法，不是附加模块。目标是“让社会自己在运行”。

#### 虚拟空间实体定义（v1.0 落地）

世界地图已扩展为可演化开放区块系统，固定区块包含但不限于：

- 生活与保障：住宅区、安宁公地、静默角落、修复站  
- 教育成长：幼儿园、小学、中学、大学  
- 劳动与生存：办公楼、工厂、农场、动物照护空间  
- 治理与公共：公开广场、商业区、娱乐设施、法庭  
- 生态：公园、植物园、动物园  

每个实体都具备地理位置信息（用于世界坐标渲染）和角色标签（`公开/协作/支撑/修复/治愈`），决定分身迁徙和行为偏好。

#### 生命闭环（从新生到离世）

- 出生期：默认在“接生医院”区进入婴幼儿阶段；新生仅允许低压行为，避免社会主导压力。  
- 幼儿/少儿/青少年：会在幼儿园与学校之间切换，优先协作学习与规则认知。  
- 青年与成体：进入上学/上班日程，出现“提案—协作—休息”的日内循环。  
- 中老年：偏向居住区/公园/修复站，承担社会共识与安抚职责。  
- 高龄：当年龄达到寿限后转入安宁公地，标记为离世记忆体，停止行为输出但保留日志。

生命节律由 `ageAdvancePerTurn`、`birthCadenceTurns`、`deathRateBias` 控制，确保社会具有持续人口补给与更新。

#### GPT-image2 资产提示词基线（卡通·人文关怀）

统一风格：`warm digital cartoon, soft rounded edges, humanistic, emotional clarity, gentle color, emotional clarity, cozy ambient lighting`  

- 新人类（新生）：卡通婴儿形象，安全、柔和、宽容色温  
- 儿童/少儿：温暖课堂与公共学习场景  
- 青年：城市化但温柔的卡通写实比例，减少压力化表情  
- 成人：可感知社会角色、保留表情与姿态信息  
- 长者：稳重、慈善感、低饱和色并强调尊严  
- 区块示例：医院、学校、办公楼、商区、农场、公园、动物园、植物园、静默角落、安宁公地均提供对应提示词模板，作为未来卡通化图像资源统一入口。

##### 细化拟人资产清单（开放世界地图）

- 人员类：新生婴儿、幼童、少年、青年、成年人、长者、职业化分身（医生、老师、律师、法官、程序员、建筑师、设计师、白领、工人、农民等），每类对应稳定风格参数：`warm digital cartoon` + soft rounded 轮廓 + 高可读表情。
- 场景类：接生医院、幼儿园、小学、中学、大学、住宅区、办公楼、工厂、农场、法院、公共广场、商业区、娱乐区、动物园、植物园、静默角落、修复站、安宁公地、夜市、公园。
- 生态类：犬猫、鸟类、牛羊马、树木、花卉、草坪、溪流鱼群。  
  风格统一为“人文关怀 + 情绪可见 + 低刺激色温”，用于后续一键批量生成。

**世界循环**

- 输入：用户用自然语言输入现实事件或关系变化。  
- 解释：系统将输入转为“社会事件种子”。  
- 执行：分身根据人格与关系做自动行动。  
- 反馈：事件流展示发生了什么、谁受影响、关系如何变化。  
- 沉淀：形成社会回声，写入分身记忆，供后续轮次引用。  

**游戏化设计（轻量）**

- 不是高指标刷怪，而是“关系体验驱动”：
  - 有“协作提案”、“倾听回应”、“公开协商”等少量行为类型。  
  - 有“短任务卡”作为体验引导，如“为低情绪分身做一次安抚回应”“完成一次可见的公开沟通”。  
  - 任务完成后产生叙事结果，不强调排行榜。

#### 游戏化增长目标

- **探索型目标**：改变自然语言场景，观察不同世界预设下行为分布的变化。  
- **治理型目标**：维持社会准则评分稳定，提高平等与开放指数。  
- **关系型目标**：通过漂流瓶、现实输入和玩家动作持续修复高风险关系链。  
- **成长型目标**：关注“区域协作覆盖率”“任务完成率”“低张力回合率”，形成可复盘成长线。

### 7.2 人生剧本（世界种子系统）

人生剧本不再是一次性填写，而是“持续生长的世界地图”。  
用户可逐步补充：

1. 关键关系（家人、同事、伴侣、朋友、边界）。  
2. 最近事件片段与当时情绪。  
3. 你常用的表达方式。  
4. 禁区与脱敏规则。  

### 7.3 AI 镜像分身（社会智能代理）

分身能力升级为“理解/参与/复盘”三类：

1. 理解：复述用户世界状态、识别事件触发点。  
2. 参与：以分身身份发起/响应行为，参与社会行动。  
3. 复盘：在回声层面解释为何做出此反应，形成稳定成长线。  

三种模式仍保留：镜子、旁观、陪伴，用于不同阶段的社会情境。

### 7.4 平行时空灵魂对话

在分身世界中启动一轮对话不是“答题”，而是进入一个可触发社会行动的镜像场景：

1. 读入最近事件与分身状态。  
2. 用户选择“观察 / 问责 / 试错 / 退一步”。  
3. 对话结束后自动生成社会级回声（行动关系摘要 + 你想继续的下一步）。

### 7.5 交换人生（他者视角观察）

交换人生仍以授权为前提，但重心从“情节体验”转到“社会行为理解”：

1. 仅在边界内允许进入他者分身。  
2. 强制匿名与重构。  
3. 体验回传只以可复用洞见输出，不直接导向人际干预。  

### 7.6 困惑交换

困难困惑通过“世界行动建议”返回：  
同一困惑给出“个体内心线索 / 社会互动线索 / 行动边界线索”，用户可选择只听、仅看或继续执行。

### 7.7 精神漂流瓶（弱连接）

漂流瓶是社会中的“偶遇事件”，强调出现时机和关系共鸣，而非高频推送：

1. 用户投放一句话或状态片段。  
2. 系统匹配低打扰共鸣场景。  
3. 接收者可只读、回应或放回。  
4. 高风险内容进入危机支持分流，不走常规漂流。  

### 7.8 回家模式

社会运行可能很复杂，回家模式提供 3 分钟人机整理层：

1. 今天发生了什么（高可信摘要）。  
2. 你最需要的不是答案，而是一个“明天可做的动作”。  
3. 系统帮助你把回声转为现实可执行计划。

## 8. 用户流程

### 8.1 首次进入

1. 输入一个当下困惑作为种子。  
2. 完成最小人生剧本。  
3. 分身被创建并进入默认开放世界。  
4. 观察第一轮社会事件回声，确认系统理解是否对齐。  

### 8.2 日常参与

1. 用户补充现实片段（越短越可持续）。  
2. 社会世界在后台自动运转。  
3. 用户可主动发起一次社会行为（提案、协作、倾听、安抚）。  
4. 回家模式整理今日回声。  

### 8.3 情绪低谷

1. 识别高风险先行兜底。  
2. 低压场景启用“允许脆弱”规则（降低互动要求，提升安抚路径）。  
3. 用户可只接收“安全回声”，暂不参与社会动作。

## 11. 技术架构

### 11.1 数据层

1. 人生事件：文本、语音转写、时间、关系、风险标签。  
2. 社会状态：分身情绪/关系/信任值。  
3. 行为日志：事件类型、影响对象、结果原因。  
4. 安全边界：禁区、撤回记录、危机优先级。

### 11.2 AI 层

1. 事件解析：从自然语言抽取场景、关系、情绪。  
2. 行为决策：对社会秩序友好、可复盘的行动候选。  
3. 回声生成：生成“可执行结论+关系影响说明”。  
4. 危机识别：高风险语义识别与降噪分流。  

### 11.3 运行层

1. 世界状态引擎（回合推进）。  
2. 任务引导模块（轻量任务卡）。  
3. 可视化事件流与社会看板。  
4. 回声归档与回家模式摘要接口。

### 11.4 Agent 推演层

1. citizen agent 负责分身的行动候选、情绪响应和回声解释。  
2. 规则审计 agent 负责检查自由、平等、开放与脆弱保护是否被破坏。  
3. 记忆整理 agent 负责把事件流写入短期、中期、长期记忆。  
4. 叙事 agent 负责把世界变化转成用户可读的回声。  
5. 世界调度器负责推进回合并裁决最终状态。

## 12. 隐私、安全与伦理边界

### 12.1 隐私原则

1. 用户拥有新增、编辑、撤回与删除权。  
2. 默认匿名与脱敏。  
3. 未经授权不得导出至交换体验。  
4. 清晰标注 AI 生成与用户输入边界。  

### 12.2 心理与社交安全

1. 不替代精神卫生专业干预。  
2. 对高风险内容触发分流，不强制交互。  
3. 脆弱状态不会被判定为“异常活跃性差”；默认转向支持性交互。  
4. 强者角色不能形成长期表达垄断。

### 12.3 社会治理

1. 公开反馈通道透明化。  
2. 越界行为留痕并可申诉。  
3. 持续修复机制：对高冲突回合增加对话引导，而不是惩罚性惩处优先。  

## 13. 商业模式

### 13.1 订阅

1. 记忆保留更长。  
2. 更丰富的社会规则模板。  
3. 更深层的回声归档与复盘。  
4. 社会导览与家庭沉淀时长。  

### 13.2 其他

硬件与服务能力后置，但预留与 App/音箱的高耦合入口。

## 14. 关键指标

### 14.1 体验指标

1. 连续输入率（是否连续补充现实片段）。  
2. 社会回声理解率（用户是否认为可复盘）。  
3. 回家模式完成率。  
4. 低负担参与率（脆弱时段仍可参与的比例）。  
5. 主观被理解感分值。  

### 14.2 关系指标

1. 关系线索更新率。  
2. 支持性回应占比。  
3. 社会冲突降温速度。  
4. 撤回/不适感投诉率。  

### 14.3 安全指标

1. 风险识别准确率。  
2. 越界事件处理及时率。  
3. 隐私撤回执行时延。  
4. 用户对社会准则理解度。  

## 15. 品牌与体验调性

以“人文社会冒险”而非“竞技竞技场”为基调。节奏克制、关系可见、情绪有容纳、世界可持续，不追求外显刺激，而追求内在回响。

关键词：

## 17. 典型场景样例

### 17.1 职业迷茫

林夏写下：  
“我今天又在会议里被忽略，我没再反驳，回去后一直自责。”  

系统将其转为世界事件：同事关系边缘化。  

分身在社会层做了两种行动：一次“发起协商提案”，一次“向另一分身求助”。  
事件流显示她获得了一个支持回应：“先把事实写清，下一步只做一次边界表达。”  
回家模式给出明天动作：写下 3 行事实版本并做一次预演。

### 17.2 关系脆弱

用户写下：  
“我不知道是不是我太敏感，但我总等别人先说。”  

分身先进入“允许脆弱”路径，先以倾听行为稳定情绪；随后生成一个公开/私域表达建议。  
用户没被要求立刻“闯任务”，而是看见“安全表达”也会得到系统反馈。

### 17.3 回到家后

回家时的摘要显示：  
“你今天的社会参与以观察为主，已完成一次有效情绪降噪。你可以选择：只记录、复盘提问、或让明天做一个边界小行动。”

## 19. 最小可讲述 Demo

用户今天输入：“今天又在群里被当作沉默者处理，我不敢发言。”  

系统先生成社会事件：公开讨论中的参与失衡。  
分身并自动发起一次“边界提问提案”：  
“我希望下一轮先表达一次我对这件事的观察。”  

用户在回声里看见：
- 同理分身做了安抚回应；  
- 一个支持分身提供了复盘模板；  
- 冲突温度下降。  

回到家后，系统只给明天一个动作：  
“发一段 80 字事实陈述，不带评价。并记录一句‘我想被听见’。”

这段体验表达了产品核心：现实片段驱动社会、社会自动运行、规则保护脆弱、并能回家转化为行动。

## 20. 产品边界总结

镜像人生不追求热闹指数，而追求“真实可复盘”。  
它的目标是：  

1. 让每个用户的分身都被持续理解；  
2. 让脆弱有安全处置路径；  
3. 让“强者”成为可依托的协作文化，而非排他权力。
